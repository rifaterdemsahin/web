To open Google Chrome windows on specific monitors using PowerShell, we need to use a different approach because PowerShell itself doesn't directly support monitor targeting for application windows. One common method is to use additional scripting or third-party utilities to move windows after they have been opened.

Here, I will outline a PowerShell script that uses the Set-WindowPosition function from the PInvoke User32.dll library to position the Chrome windows after they are opened. This script assumes you have multiple monitors configured horizontally (side by side).

PowerShell Script with Monitor Targeting

Add-Type @"
using System;
using System.Runtime.InteropServices;

public class User32 {
                    [DllImport("user32.dll", SetLastError = true)]
                    public static extern bool MoveWindow(IntPtr hWnd, int X, int Y, int nWidth, int nHeight, bool bRepaint);
                
                    [DllImport("user32.dll", SetLastError = true)]
                    public static extern IntPtr FindWindow(string lpClassName, string lpWindowName);
                }

"@

Define the URLs and monitor positions (0 for primary, 1 for secondary, etc.)

$urls = @(
@{ URL = "https://www.example.com"; Monitor = 0 },
@{ URL = "https://www.google.com"; Monitor = 1 },
@{ URL = "https://www.openai.com"; Monitor = 0 }
)

Assume 1920x1080 resolution for simplicity; adjust as necessary for your monitors

$width = 1920
$height = 1080

Open each URL in a new Chrome window and move it to the specified monitor

foreach ($item in $urls) {
$process = Start-Process "chrome.exe" "--new-window $($item.URL)" -PassThru
Start-Sleep -Seconds 2 # Wait for Chrome to load

$window = [User32]::FindWindow(null, "New Tab - Google Chrome")
                if ($window -ne [IntPtr]::Zero) {
                    $xPosition = $item.Monitor * $width
                    [User32]::MoveWindow($window, $xPosition, 0, $width, $height, $true)
                }

}

How This Script Works

  • PInvoke Setup: This script begins by adding type definitions with PInvoke to use the MoveWindow and FindWindow functions from the User32.dll. This allows controlling window positioning programmatically.

  • URL and Monitor Definitions: URLs are stored in a hashtable with a key for the URL and a monitor index (0 for the primary monitor, 1 for the second, etc.).

  • Window Moving: For each URL, Chrome is started, and the script waits briefly to ensure the window has loaded. The FindWindow function looks for the window titled "New Tab - Google Chrome". Once found, the script uses the MoveWindow function to place the window on the designated monitor based on the index multiplied by the width of a monitor (assuming all monitors have the same resolution).

Adjustments You Might Need

  • Resolution: Adjust the $width and $height variables to match the resolution of your monitors.

  • Delay: The Start-Sleep may need adjustment depending on how quickly Chrome launches on your system.

  • Window Title: The window title used in FindWindow must match exactly; it might change based on the active tab or page title.

This script assumes a relatively simple setup with monitors having the same resolution. If your configuration differs, adjustments to the script will be necessary.

Certainly! Below is the revised AutoHotkey script that should correctly open each URL in a new Google Chrome window on the designated monitor:

; Define URLs and corresponding monitors (0 = primary, 1 = secondary, etc.)
urls := ["https://www.example.com", "https://www.google.com", "https://www.openai.com"]
monitors := [0, 1, 0]

; Assume 1920x1080 resolution for simplicity; adjust as necessary for your monitors
ScreenWidth := 1920
ScreenHeight := 1080

Loop, % urls.Length()
{
; Launch Chrome with the specified URL
Run, % "chrome.exe --new-window " urls[A_Index]
WinWait, ahk_exe chrome.exe, , 10 ; Wait up to 10 seconds for the window to appear

; Calculate the position based on the monitor index
                XPos := monitors[A_Index] * ScreenWidth
                
                ; Move the Chrome window to the specified monitor
                IfWinExist, ahk_exe chrome.exe
                {
                    WinActivate ; Make sure the window is active
                    WinMove, A,, XPos, 0, ScreenWidth, ScreenHeight ; Move and resize the window
                }
                Sleep 1000 ; Wait a moment before opening the next window to prevent overlaps

}

To use this script:

  • Copy the above script into a text editor.

  • Save the file with a .ahk extension (for example, OpenChromeWindows.ahk).

  • Double-click the .ahk file to run the script if you have AutoHotkey installed.

This script includes an additional Sleep command at the end of the loop to ensure that windows do not overlap if they are created too quickly in succession. Adjust the ScreenWidth and ScreenHeight to match the resolution of your monitors. If your monitor setup has different resolutions or a different arrangement, you may need to adjust the calculations for XPos.
