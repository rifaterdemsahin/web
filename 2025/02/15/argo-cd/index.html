The "Back-off restarting failed container" error typically indicates that a container in a pod is repeatedly failing to start and Kubernetes is backing off from trying to restart it too frequently. In your case, this is happening with the dex container in the argocd-dex-server pod.

Step 1: Check Pod Logs

Step 2: Describe the Pod

  • Describe the pod:

oc describe pod argocd-dex-server-69bb969bd7-7gt7k -n argocd

The dex container is part of the Argo CD's identity management system and is responsible for handling authentication requests. Issues with the configuration, such as incorrect OAuth settings or problems with connecting to an external identity provider, can cause the dex container to fail.

oc get configmap argocd-cm -n argocd -o yaml

  • Delete the failing pod to force a restart:
