It sounds like you're encountering several issues in the Event Viewer that could be causing system instabilities or errors. Here’s a breakdown of the steps to address each issue:

1. Secure Boot Error:

  • This error typically appears if Secure Boot is not enabled in the system’s BIOS or UEFI settings.

  • To resolve this, follow these steps:

Restart your computer and access the BIOS/UEFI (usually by pressing keys like F2, F10, DEL, or ESC during startup).

  • Look for a Secure Boot option in the Security or Boot settings.

  • Enable Secure Boot and save the settings.

  • Restart your computer to apply the changes.

  • If Secure Boot is already enabled, try toggling it off, restarting, then enabling it again.

2. COM Server Application Permissions (CLSID & APPID Issues):

  • These errors relate to application-specific permissions that can be modified using Component Services.

  • Here’s how to grant Local Activation permission:

Open Component Services by searching for it in the Start menu.

  • In the left pane, go to Component Services > Computers > My Computer > DCOM Config.

  • Locate the CLSID or APPID given in the error log (you may have to match the CLSID or APPID numbers manually).

  • Right-click the application, select Properties, then go to the Security tab.

  • In Launch and Activation Permissions, click Edit, then add the user or group specified in the error (like AzureAD\Pexabo or LOCAL SERVICE), and grant Local Activation permission.

  • Click OK to save the changes and restart the system if necessary.

3. Ephemeral Port Exhaustion Error:

  • This error suggests that all available UDP ports are in use, often caused by a high number of active connections or applications holding ports open.

  • To resolve this:

Restart the computer to release all ephemeral ports.

  • Use the command prompt to check active ports with netstat -ano | findstr :<port_number>.

  • If specific applications are monopolizing ports, consider restarting or reconfiguring those services.

  • Increase the ephemeral port range:

Open a command prompt as an administrator.

  • Run the command:
    shell netsh int ipv4 set dynamicport udp start=1024 num=64512

4. Smart Card Logon Certificate Error:

  • If you’re not using a domain-joined computer or if there are no issues with smart card logon, this error can be safely ignored.

  • Otherwise, you may need to install the appropriate certificate on the client machine, matching the requirements of the domain.

5. Network Link Disconnection (Intel Ethernet Controller X550):

  • This error often indicates a physical disconnection, a driver issue, or a problem with the network cable.

  • Check the following:

Ensure the network cable is securely connected.

  • Update the network driver by going to Device Manager > Network adapters > Intel(R) Ethernet Controller X550. Right-click and select Update driver.

  • Test a different network cable or port on the router if available.

Address each error one at a time, starting with the highest priority based on your system needs. If issues persist after these steps, some could be safely ignored, especially if they do not impact system performance.
